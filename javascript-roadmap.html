📚 Phase 1: Core JavaScript Fundamentals

3-4 weeks
Variables, Data Types & Type Coercion

What to learn: let vs const vs var, primitives vs objects, truthy/falsy values, type conversion vs coercion

🔨 Build This: A Shopping Cart Calculator

  • User inputs: item name (string), price (could be "19.99" or 19.99), quantity (string or number)
  • Handle edge cases: empty strings, "0", null, undefined
  • Calculate total without using Number() or parseInt() - let coercion work
  • Apply discount code (string "SAVE10" = 10% off)
  • Return formatted output: "Total: $89.91 (3 items)"

💡 You'll discover how "5" + 3 vs "5" - 3 behaves!

Guide MDN Docs
Functions Deep Dive

What to learn: Function declarations vs expressions, arrow functions, parameters vs arguments, return values, default parameters

🔨 Build This: A Text Formatter Toolkit

  • Create formatTitle(text) - capitalizes first letter of each word
  • Create shortenText(text, maxLength = 50) - adds "..." if too long
  • Create countWords(text) - returns word count
  • Create a higher-order function: applyFormat(text, formatter) - takes a function as argument
  • Chain them: shortenText(formatTitle("hello world"), 20)

💡 You'll learn the difference between function types and when to use each!

Scope & Closures

What to learn: Global vs local scope, function scope vs block scope, lexical scope, closure concept and practical uses

🔨 Build This: A Bank Account System

  • createAccount(initialBalance) - returns object with deposit(), withdraw(), getBalance()
  • Balance should be PRIVATE (not accessible from outside)
  • Keep transaction history inside the closure
  • Example: const myAccount = createAccount(100); myAccount.deposit(50); myAccount.getBalance(); // 150
  • Try to access balance directly - it shouldn't work!

💡 You'll see how closures create truly private data!

Guide
Arrays & Objects Mastery

What to learn: Array methods (map, filter, reduce, forEach, find), object manipulation, destructuring, spread/rest operators

🔨 Build This: A Student Grade Analyzer

  • Start with: students = [{name: "Alice", grades: [85, 92, 88]}, {name: "Bob", grades: [70, 75, 80]}]
  • Use .map() to add an 'average' property to each student
  • Use .filter() to find students with average > 80
  • Use .reduce() to calculate class average
  • Use .find() to get top student
  • Use destructuring to extract student data cleanly
  • NO for loops allowed - only array methods!

💡 You'll master data transformation - the most important JS skill!

this Keyword & Context

🔨 Build This: A Game Character System

  • Create a character object: {name: "Hero", health: 100, attack() {...}}
  • attack() method should use 'this' to access character's name
  • Create takeDamage(amount) that reduces health using 'this'

Callbacks & Callback Hell

What to learn: What callbacks are, callback patterns, understanding callback hell problem

🔨 Build This: Recipe Step-by-Step Timer

  • Simulate cooking steps with callbacks:
  • boilWater(callback) → addPasta(callback) → drain(callback) → serve()
  • Each step takes different time (use setTimeout)
  • Log each step: "Boiling water...", "Adding pasta...", etc.

Promises from Scratch

What to learn: Promise states, then/catch/finally, promise chaining, Promise.all/race/allSettled

🔨 Build This: Image Loader with Promises

  • Create loadImage(url) that returns a Promise
  • Resolve when image loads successfully, reject on error
  • Chain multiple images: loadImage(url1).then(() => loadImage(url2)).then(...)
  • Use Promise.all() to load 3 images at once
  • Compare Promise.all vs Promise.race - load 5 images, see which finishes first
  • Handle errors with .catch() - try with broken image URL

💡 You'll see how Promises clean up callback hell!

Guide
Async/Await Mastery

What to learn: How async/await works, error handling with try/catch, parallel vs sequential execution

🔨 Build This: Fake API Data Dashboard

  • Create fake async functions: getUser(), getPosts(), getComments() (use setTimeout to simulate delay)
  • Fetch them SEQUENTIALLY with async/await - measure total time
  • Refactor to run them in PARALLEL - compare speed!
  • Add try/catch to handle errors gracefully
  • Example code structure: 
    async function loadDashboard() {
  try {
    const user = await getUser();
    const [posts, comments] = await Promise.all([
      getPosts(user.id), 
      getComments(user.id)
    ]);
    console.log({ user, posts, comments });
  } catch (error) {
    console.error('Failed:', error);
  }
}

💡 You'll learn when to run things in parallel vs sequential!

🌐 Phase 4: DOM & Browser APIs

2 weeks
DOM Manipulation Fundamentals

What to learn: querySelector, createElement, appendChild, innerHTML vs textContent, classList

🔨 Build This: Dynamic Product Card Generator

  • Start with array: products = [{name: "Laptop", price: 999, inStock: true}, ...]
  • For each product, CREATE elements (don't use innerHTML template)
  • Build structure: div.card > h3 > p.price > button
  • Add "out of stock" class if inStock is false
  • Add click event to button (logs "Added to cart")
  • NO frameworks, NO templates - pure DOM manipulation!

💡 You'll understand what React does under the hood!

Event Handling Deep Dive

What to learn: Event bubbling/capturing, event delegation, preventDefault, custom events

🔨 Build This: Interactive Comment Section

  • Create nested comment structure (comments with replies)
  • Use EVENT DELEGATION - attach ONE listener to parent, not each comment
  • Clicking "Reply" shows text input under that comment
  • Clicking "Delete" removes comment (use event.target to find which one)
  • Add "Like" button - prevent event from bubbling up
  • Test: console.log which phase the event is in (bubbling vs capturing)

💡 You'll see why event delegation is powerful for dynamic content!
