1.3.5 - For Loops: Dice Roller

What you're learning: For loops - how to repeat code a specific number of times

🔗 You'll use loops to roll multiple dice in your final app

🎲 Project: Multi-Dice Roller

What to build: A function that rolls multiple dice and stores results

Understanding for loops first:

A for loop has 3 parts:

for (let i = 0; i < 5; i++) {
  // Code here runs 5 times
}

Part 1: let i = 0 - Start counting at 0

Part 2: i < 5 - Keep going while i is less than 5

Part 3: i++ - Add 1 to i after each loop

What happens:

  • Loop 1: i = 0, run the code
  • Loop 2: i = 1, run the code
  • Loop 3: i = 2, run the code
  • Loop 4: i = 3, run the code
  • Loop 5: i = 4, run the code
  • Stop (because 5 is NOT less than 5)

Think of i as: "Which loop number am I on?"

Exact steps:

  1. Create file: diceRoller.js
  2. Write a function: rollMultipleDice(numberOfDice, sides)
    • numberOfDice = how many dice to roll (like 3)
    • sides = what kind of dice (like 20 for d20)
  3. Inside the function:
    • Create an empty array: let rolls = []
    • Make a for loop that runs numberOfDice times
    • Inside the loop: roll one die and push to the array
    • To roll one die: Math.floor(Math.random() * sides) + 1
  4. After the loop, return the rolls array

Example:
rollMultipleDice(3, 20) might return: [15, 8, 19]
rollMultipleDice(2, 6) might return: [4, 2]
(Your results will be random!)

Step-by-step example of the loop:
If numberOfDice = 3:

  • Loop 1 (i=0): Roll die, get 15, push to array → [15]
  • Loop 2 (i=1): Roll die, get 8, push to array → [15, 8]
  • Loop 3 (i=2): Roll die, get 19, push to array → [15, 8, 19]
  • Stop (i=3 is NOT < 3)
  • Return [15, 8, 19]

✅ Done when:
• rollMultipleDice(3, 20) returns array with 3 random numbers
• rollMultipleDice(5, 6) returns array with 5 random numbers
• You understand what i represents in the loop

🎲 Project: Promise-Based Dice Roller

What to build: A function that rolls dice with a delay, using Promises

First: Learn what a Promise is

A Promise represents something that will complete in the future

It's like saying "I promise to give you the result when I'm done"

return new Promise((resolve) => {
  // Do some work
  // When done, call: resolve(result)
});

Then you use .then() to get the result when it's ready

Part 1: Build the rollDice function

  1. Create file: diceRoller.js
  2. Write function signature: function rollDice(sides, count = 1)
  3. Inside the function: return new Promise((resolve) => {
    • The word new is required before Promise
    • resolve is the parameter - this is the function you'll call when done
  4. Inside the Promise: setTimeout(() => { with 500 as the delay
  5. Inside the setTimeout:
    • Create an empty array called rolls
    • Write a for loop that runs count times
    • Inside the loop: generate a random number from 1 to sides, push it into rolls
    • AFTER the for loop: create a variable called total
    • Use .reduce() on the rolls array to calculate the total
    • Call resolve() with an object containing three properties:
      • rolls: rolls (the array)
      • total: total (the sum)
      • formula: count + "d" + sides (string like "2d20")
  6. Close all brackets: }, 500); then }); then }

Part 2: Test with 4 tasks

  1. Task 1: Roll 1d20 and log the result
    • Call rollDice(20) and use .then() to log the result
    • You should see an object with rolls, total, and formula
  2. Task 2: Chain two rolls - attack then damage

    • Second: AFTER you get the attack result, roll 2d6 for damage

    • Log both the attack result and damage result
    • Hint: You'll need to chain .then() calls
  3. Task 3: Roll 3 characters' initiative simultaneously
    • Roll 1d20 three times - but all at the SAME TIME (not one after another)
    • Use Promise.all() to run multiple promises at once
    • You'll get back an array containing all 3 results
    • Log all 3 initiative rolls
  4. Task 4: Implement advantage (roll 2d20, keep higher)
  5. Roll two d20s at the same time
  6. Compare the totals of both rolls
  7. Keep whichever one is higher
  8. Log the winning roll
  9. Hint: Promises can run in parallel with Promise.all

💡 Key syntax requirements:
- Must use new Promise (not just Promise)
- Must call resolve() when done
- Must create rolls array before the for loop
- Must calculate total after the for loop

2.3 - Async/Await: Attack Roll System

What you're learning: async/await, try/catch, parallel vs sequential

🔗 Uses the dice roller from 2.2 in cleaner syntax

⚔️ Project: Complete Attack System

What to build: Attack resolver using async/await (cleaner than .then())

First: Learn what async/await does

async/await is a cleaner way to work with Promises

Instead of .then(), you use await

// With .then():
rollDice(20).then(result => console.log(result));

// With async/await:
async function roll() {
  const result = await rollDice(20);
  console.log(result);
}

await means "wait for this Promise to finish before continuing"

Exact steps:

  1. Create file: attackSystem.js
  2. Copy your entire rollDice function from 2.2 into this file
  3. Create function: async function makeAttack(attackBonus, damageBonus, targetAC)
    • Must have async keyword before function
    • Takes three parameters: attack bonus, damage bonus, target AC
  4. Wrap your entire function logic in try { } catch (error) { }
    • In the catch: log the error and return null
  5. Inside the try block, build the attack logic:
    • Step 1: Roll a d20 using await rollDice(20), store the result
    • Step 2: Calculate the total attack roll (d20 result + attackBonus)
    • Step 3: Check if the attack hits (compare to targetAC)
    • If HIT: Roll damage (2d6 using rollDice), add damageBonus, return success object
    • If MISS: Return miss object
    • Your return objects need properties: hit (boolean), damage (if hit), attackRoll (total)
  6. Test your function: makeAttack(5, 3, 15).then(result => console.log(result))

💡 Key concepts:
- Use await before EACH rollDice call
- Attack roll happens FIRST, damage ONLY if hit (sequential)
- async functions automatically return Promises
- try/catch handles errors from await

✅ Done when: Attack system correctly rolls, compares AC, and handles hits/misses
