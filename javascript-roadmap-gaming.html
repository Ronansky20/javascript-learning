1.3.5 - For Loops: Dice Roller

What you're learning: For loops - how to repeat code a specific number of times

🔗 You'll use loops to roll multiple dice in your final app

🎲 Project: Multi-Dice Roller

What to build: A function that rolls multiple dice and stores results

Understanding for loops first:

A for loop has 3 parts:

for (let i = 0; i < 5; i++) {
  // Code here runs 5 times
}

Part 1: let i = 0 - Start counting at 0

Part 2: i < 5 - Keep going while i is less than 5

Part 3: i++ - Add 1 to i after each loop

What happens:

  • Loop 1: i = 0, run the code
  • Loop 2: i = 1, run the code
  • Loop 3: i = 2, run the code
  • Loop 4: i = 3, run the code
  • Loop 5: i = 4, run the code
  • Stop (because 5 is NOT less than 5)

Think of i as: "Which loop number am I on?"

Exact steps:

  1. Create file: diceRoller.js
  2. Write a function: rollMultipleDice(numberOfDice, sides)
    • numberOfDice = how many dice to roll (like 3)
    • sides = what kind of dice (like 20 for d20)
  3. Inside the function:
    • Create an empty array: let rolls = []
    • Make a for loop that runs numberOfDice times
    • Inside the loop: roll one die and push to the array
    • To roll one die: Math.floor(Math.random() * sides) + 1
  4. After the loop, return the rolls array

Example:
rollMultipleDice(3, 20) might return: [15, 8, 19]
rollMultipleDice(2, 6) might return: [4, 2]
(Your results will be random!)

Step-by-step example of the loop:
If numberOfDice = 3:

  • Loop 1 (i=0): Roll die, get 15, push to array → [15]
  • Loop 2 (i=1): Roll die, get 8, push to array → [15, 8]
  • Loop 3 (i=2): Roll die, get 19, push to array → [15, 8, 19]
  • Stop (i=3 is NOT < 3)
  • Return [15, 8, 19]

✅ Done when:
• rollMultipleDice(3, 20) returns array with 3 random numbers
• rollMultipleDice(5, 6) returns array with 5 random numbers
• You understand what i represents in the loop

2.2 - Promises: Dice Roll System

What you're learning: Promises, .then(), .catch(), Promise.all

🔗 This dice roller will be the core of your character sheet's actions!

🎲 Project: Promise-Based Dice Roller

What to build: Async dice rolling with animation delays

Exact steps:

  1. Create file: diceRoller.js
  2. Write function that returns a Promise: 
    function rollDice(sides, count = 1) {
  return new Promise((resolve) => {
    setTimeout(() => {
      const rolls = [];
      for (let i = 0; i < count; i++) {
        rolls.push(Math.floor(Math.random() * sides) + 1);
      }
      const total = rolls.reduce((sum, roll) => sum + roll, 0);
      resolve({ rolls, total, formula: count + "d" + sides });
    }, 500); // 500ms "animation" delay
  });
}
  3. Task 1: Roll 1d20 and log result
  4. Task 2: Chain: Roll 1d20 for attack, THEN 2d6 for damage
  5. Task 3: Use Promise.all to roll 3 characters' initiative at once
  6. Task 4: Add advantage (roll 2d20, take higher)

Example usage:
rollDice(20).then(result => console.log("Rolled:", result.total));
Promise.all([rollDice(20), rollDice(20), rollDice(20)])

✅ Done when: Can roll dice with delays & chain multiple rolls

2.3 - Async/Await: Attack Roll System

What you're learning: async/await, try/catch, parallel vs sequential

🔗 Uses the dice roller from 2.2 in cleaner syntax

⚔️ Project: Complete Attack System

What to build: Attack resolver using async/await

Exact steps:

  1. Create file: attackSystem.js
  2. Import your rollDice function from 2.2
  3. Write async function: 
    async function makeAttack(attackBonus, damageBonus, targetAC) {
  // Roll attack (d20 + bonus)
  const attackRoll = await rollDice(20);
  const attackTotal = attackRoll.total + attackBonus;
  
  if (attackTotal >= targetAC) {
    // Hit! Roll damage
    const damageRoll = await rollDice(6, 2); // 2d6
    const totalDamage = damageRoll.total + damageBonus;
    return { hit: true, damage: totalDamage, attackRoll: attackTotal };
  } else {
    return { hit: false, attackRoll: attackTotal };
  }
}
  4. Add try/catch for error handling
  5. Compare sequential vs parallel:
    • Sequential: Attack then damage (one after another)
    • Parallel: Roll 3 party members' attacks at once

✅ Done when: Attack system works & you understand when to use parallel
